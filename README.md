## What Is Provably Fair?

Provably fair is a cryptographic method that allows players to verify that game outcomes were not manipulated. It uses **HMAC-SHA256** hashing to generate game results from three inputs:

- **Server Seed** — a random string generated by the casino
- **Client Seed** — a string you set yourself
- **Nonce** — a counter that increments with each bet

The server commits to a seed hash *before* the bet. After the bet, the server seed is revealed so you can verify the hash matches and recalculate the result independently.

## How Verification Works

1. Before your bet, the casino shows you a **hashed server seed** (SHA-256)
2. You can set your own **client seed** and see the **nonce** (bet counter)
3. After revealing the server seed, you can verify:
   - The hash of the revealed server seed matches what was shown before
   - The game result matches the HMAC-SHA256 calculation

## Technical Details

The core algorithm for most games:

1. Generate an HMAC-SHA256 hex string using the server seed as key and `clientSeed:nonce` as the message
2. Take the first 8 hex characters and convert to an integer
3. Divide by 4,294,967,295 to get a float between 0 and 1
4. Map that float to the specific game outcome (dice roll, crash point, etc.)

Each game type uses a different mapping function, but the randomness source is identical.
